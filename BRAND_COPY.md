## 1. 品牌定位

**一句话定位**：

```text
Local-first, privacy-first, report-grade Gemini chat export.
本地优先、隐私优先、出版级排版的 Gemini 聊天导出工具。
```

**三大差异化**：

1. **隐私可信，而不是只说隐私**
   - 所有解析与渲染均在浏览器本地完成
   - 聊天正文绝不上传到任何服务器
   - PDF / Word / PNG 全部本地生成，不使用自有服务器转档

2. **出版级文档，而不是普通截图/普通 PDF**
   - Word / PDF 输出像正式报告，而不是把网页粗暴打印出来
   - 支持封面、目录、章节层级、代码块、表格、图片说明、来源信息
   - Pro 主题围绕真实场景命名：Academic Paper、Business Report、Research Brief、Client Memo、Newsprint

3. **一键报告化，而不是只导出聊天**
   - AI-only Report：去掉用户 Prompt，只导出 AI 输出
   - Q&A Transcript：保留用户和 AI 的问答结构
   - Single Answer Card：只导出某一条回复为长图或卡片
   - Research Archive：保留时间、来源 URL、平台信息

**支持平台**：Gemini (gemini.google.com)

**导出格式**：PDF、Word、Markdown、PNG 图片、HTML、TXT、JSON

## 3. 文案设计原则

### 3.1 标题（extensionName / extensionActionTitle）

**统一句式**：`Gemini Exporter - {核心差异化卖点}`

- 保留 "Gemini Exporter" 品牌名前缀，明确平台归属
- 破折号后紧跟一句话核心卖点：**隐私优先 + 本地运行 + PDF 等多格式**
- Chrome Web Store 标题字符限制约 75 字符，各语言均控制在限制内

### 3.2 描述（extensionDescription）

**三段式结构**：

1. **能力陈述**：导出 Gemini 对话到 PDF/Word/Markdown/PNG 等格式
2. **隐私承诺**：所有解析与渲染均在浏览器本地完成
3. **数据边界**：聊天正文绝不上传到任何服务器

**设计要点**：

- 不堆砌功能列表，突出"隐私优先"差异化
- 明确"本地完成"和"不上传"两个关键承诺
- 格式列表保持完整（PDF、Word、Markdown、PNG），让用户一眼看到支持范围

### 3.3 短名（extensionShortName / contextMenuExportParent / launcher_tooltip）

统一使用 "Gemini Export"，保持简洁，适合工具栏图标 tooltip 和右键菜单。
