# ChatVault Exporter - 产品需求文档 (PRD)

## 1. 产品定位与目标

**ChatVault Exporter** 是一款面向 AI 聊天平台的 **隐私优先、本地运行、出版级排版导出** Chrome 扩展。

它不是主产品 `ChatVault AI / ChatFolder AI` 的低配版，而是一个独立的轻量产品入口：不做文件夹管理、不做聊天历史索引同步、不默认保存用户聊天正文，只把当前 AI 对话稳定地导出成适合编辑、分发、归档和分享的高质量文档。

两款产品共享同一套账号与 VIP 订阅系统，实现“一次付费，多产品共享 Pro 权益”：

- `ChatVault AI / ChatFolder AI`：面向长期整理、同步、收藏、搜索和知识管理。
- `ChatVault Exporter`：面向低打扰、低权限、只在需要时导出当前对话的用户。

核心定位一句话：

```text
Local-first, report-grade AI chat export.
```

## 2. 竞品背景与差异化策略

### 2.1 参考竞品

参考竞品：`AI Exporter: Save ChatGPT, Gemini to PDF, Word, MD and Notion`

- Chrome Web Store URL: https://chromewebstore.google.com/detail/kagjkiiecagemklhmhkabbalfpbianbe
- 截至 2026-06-07 可检索页面显示：Featured、约 20,000 users、4.7 分、270 ratings；其官网自称 120,000+ users、4.8/5。Chrome Web Store 与官网数据会动态变化，后续竞品分析应以上线前复核为准。
- 主要能力：ChatGPT / Claude / Gemini / DeepSeek / Grok 等多平台导出，支持 PDF / PNG / Markdown / TXT / Word 等格式、Notion sync、Google AI Overview 导出、Markdown copy、导出推理步骤选项、多语言等。
- 其官网隐私说明提到多数能力在本地浏览器运行，但 PDF generation 会临时经过服务器处理并删除数据。这个点可以成为 `ChatVault Exporter` 的核心差异化：PDF / Word / PNG 均在浏览器本地生成，不使用自有服务器转档。

判断：竞品已经把“格式多、平台多、同步到 Notion、功能全”做得很强，`ChatVault Exporter` 不应该走单纯功能堆叠路线，否则会变成低胜率追赶。

### 2.2 差异化方向

`ChatVault Exporter` 的差异化应集中在 4 个方向：

1. **隐私可信，而不是只说隐私**
   - 默认不抓取历史列表。
   - 默认不建立本地知识库。
   - 默认不上传聊天正文到自有服务器。
   - PDF / Word / PNG 全部本地生成，不把内容发到自有服务器做转档。
   - 明确展示“本次导出涉及哪些数据、哪些会被保存、哪些不会上传”。
   - Manifest 权限尽量收窄，让用户在 Chrome 商店页也能感知安全边界。

2. **出版级文档，而不是普通截图/普通 PDF**
   - Word / PDF 输出要像正式报告，而不是把网页粗暴打印出来。
   - 支持封面、目录、章节层级、引用样式、代码块、表格、图片说明、来源信息。
   - Pro 主题围绕真实场景命名：Academic Paper、Business Report、Research Brief、Client Memo、Newsprint。

3. **一键报告化，而不是只导出聊天**
   - `AI-only Report`：去掉用户 Prompt，只导出 AI 输出，适合报告、方案、教程、总结。
   - `Q&A Transcript`：保留用户和 AI 的问答结构，适合复盘和溯源。
   - `Single Answer Card`：只导出某一条回复为长图或卡片，适合分享。
   - `Research Archive`：保留时间、来源 URL、平台、可检测到的模型信息，适合学术和工作留档。

4. **与主产品形成组合，而不是互相替代**
   - 隐私敏感用户先用 Exporter，后续需要整理/同步时自然升级到 `ChatVault AI / ChatFolder AI`。
   - 已购买主产品 Pro 的用户，在 Exporter 自动获得高级导出能力。
   - 官网和扩展内文案要清楚说明：Exporter 负责“输出”，主产品负责“管理”。

### 2.3 不建议跟随竞品做的方向

MVP 阶段不建议追逐以下功能：

- 不把 10+ 平台作为首要目标，先把 ChatGPT / Claude / Gemini 的导出质量做到稳定。
- 不把 Notion sync 作为第一优先级，先保证 Markdown / Word / PDF 的可携带性。
- 不做默认本地知识库，否则会稀释“隐私、轻量、不索引历史”的定位。
- 不用复杂侧边栏覆盖聊天页面，避免和主产品定位重叠。

## 3. 目标用户

1. **轻量需求用户**：不需要文件夹整理功能，只在需要备份、分享或排版对话时使用导出功能。
2. **隐私敏感用户**：不希望扩展同步历史聊天列表，也不希望聊天正文被上传到第三方服务。
3. **效率工作者**：需要将 AI 生成的报告、代码、长文、研究结果快速转换为 Word / PDF / PNG。
4. **研究与写作用户**：需要保留来源 URL、导出时间、平台、模型信息、引用和公式。
5. **主产品用户**：已经购买 `ChatVault AI / ChatFolder AI` Pro，希望在不打开完整管理面板时快速导出当前对话。

## 4. 核心功能需求

### 4.1 平台支持

MVP 支持：

- `https://chatgpt.com/*`
- `https://chat.openai.com/*`（历史兼容）
- `https://claude.ai/*`
- `https://gemini.google.com/*`

后续平台扩展按“导出质量优先”推进，不以平台数量作为核心卖点。

### 4.2 导出格式

1. **Word (.docx)**
   - 保留 Markdown 结构、标题层级、代码块、表格、列表、引用、图片。
   - Pro 支持封面、目录、主题样式和自定义标题。

2. **PDF**
   - 适合正式分发。
   - 支持专业主题、页眉页脚、来源信息、导出时间。

3. **PNG 图片**
   - 支持长图和单条回答卡片。
   - 针对超长内容提供分段导出，避免 canvas 过高导致失败。

4. **Markdown / TXT**
   - Free 可用基础导出。
   - Markdown 重点服务开发者、写作者和二次编辑。

5. **JSON（后续可选）**
   - 仅作为高级归档格式，不作为 MVP 必做项。

### 4.3 导出范围

- **当前会话整篇导出**：导出当前页面可读取的完整对话。
- **选择性导出**：进入选择模式，勾选特定消息轮次后导出。
- **仅导出 AI 回答**：过滤用户 Prompt，导出 AI 生成内容。
- **单条回答导出**：对单个 AI 回复生成 Word / PDF / PNG。
- **问答转录模式**：保留用户与 AI 的完整对话结构。

说明：由于本产品不做聊天历史索引同步，MVP 不承诺跨会话批量导出。若后续要做批量导出，应作为 Pro 高级能力并明确需要用户主动选择。

### 4.4 高级排版主题（Pro）

Free 仅开放 Default 主题。Pro 解锁以下主题：

- **Default Modern**：现代、清爽、通用。
- **Academic Paper**：学术论文风格，强调引用、公式、脚注和来源。
- **Business Report**：商业报告风格，适合客户方案、周报、研究简报。
- **Client Memo**：更紧凑的咨询备忘录风格。
- **Newsprint**：报纸复古风格，适合长文阅读和分享。

主题不只是颜色差异，应控制字体、字号、行距、页边距、代码块样式、引用样式、图片说明和页眉页脚。

### 4.5 导出属性自定义

基础设置：

- 显示 / 隐藏导出时间。
- 显示 / 隐藏对话标题。
- 显示 / 隐藏平台名称。
- 显示 / 隐藏角色标签。
- 附带原始 URL。

Pro 高级设置：

- 隐藏 ChatVault 水印。
- 自定义标题。
- 自定义作者 / 组织名称。
- Word / PDF 封面。
- 目录和章节编号。
- 更高长图分辨率和更大的导出上限。

### 4.6 导出前预览与可靠性

- 导出前展示格式、主题、范围、预计消息数和隐私说明。
- 长对话导出时显示加载进度。
- 图片拉取失败时保留可读 fallback 文案。
- 用户取消下载时不计入 Free 额度。
- 导出失败时展示明确原因：平台页面未加载完整、图片跨域失败、内容过长、权限缺失、扩展上下文失效等。

## 5. 账号与付费体系

### 5.1 免登录 Free

未登录用户可以直接使用基础导出：

- 每日 3 次 Free 导出。
- 仅 Default 主题。
- 基础导出设置。
- 本地记录当天使用次数。

登录不是基础导出的前置条件。登录用于：

- 激活 Pro 权益。
- 恢复购买。
- 与主产品共享订阅。
- 获取高级主题和高级导出设置。

### 5.2 共享 VIP 规则

- 同一账号在 `ChatVault AI / ChatFolder AI` 或官网购买 Pro 后，可在 `ChatVault Exporter` 使用 Pro 能力。
- Pro 判断以 Supabase 中同步后的订阅状态为准，不只依赖本地缓存。
- Pro 到期或订阅取消后，高级主题和无限导出回落到 Free 规则。

### 5.3 Free / Pro 限制

| 功能 | Free | Pro |
| :--- | :--- | :--- |
| 登录要求 | 可免登录使用基础导出 | 需要登录激活权益 |
| 每日导出次数 | 3 次 | 无限制 |
| 当前会话整篇导出 | 可用，受技术上限保护 | 可用，更高上限 |
| 选择消息导出 | 可用 | 可用 |
| 仅导出 AI 回答 | 可用 | 可用 |
| 单条回答卡片 | 可用 | 可用，高分辨率 |
| 高级主题 | 仅 Default | 全部主题 |
| 隐私证明面板 | 可用 | 可用，支持导出凭证 |
| 导出前质量检查 | 基础检查 | 完整检查与修复建议 |
| 敏感信息脱敏 | 基础规则 | 自定义规则与预设 |
| 场景化模板 | 基础报告模板 | 全部专业模板 |
| 开发者代码导出 | 基础代码保留 | 代码索引与 zip 打包 |
| 隐藏水印 | 不可用 | 可用 |
| 自定义标题 / 作者 / 封面 | 基础标题 | 高级自定义 |

## 6. UI 与交互设计

### 6.1 去管理化

移除或不实现以下能力：

- 左侧文件夹管理侧边栏。
- 标签管理。
- 聊天历史搜索。
- 自动同步聊天标题索引。
- “移入文件夹”和“添加书签”按钮。

### 6.2 非侵入式入口

- 在 AI 聊天页面右下角显示轻量 Export 按钮，使用 Shadow DOM 隔离样式。
- 入口应尽量避开平台原生输入框、发送按钮、滚动控件和底部提示。
- 支持右键菜单或扩展 action 作为备用入口。

### 6.3 导出配置面板

面板包含：

- 当前平台和对话标题。
- 登录状态与 Free 剩余额度。
- 格式选择：Word / PDF / Image / Markdown。
- 模式选择：Full Chat / AI-only / Selected / Single Answer。
- 主题选择及微缩预览。
- 导出属性开关。
- 隐私提示：聊天正文不上传到 ChatVault 自有服务器。
- 导出按钮、进度、失败原因和重试入口。

## 7. 隐私与安全

- **本地化解析**：HTML / DOM 解析、消息结构化、文档构建在浏览器本地完成。
- **不上传聊天正文到自有服务器**：导出内容不发送到 ChatVault 后端、Supabase 或支付服务。
- **图片处理说明**：为保留图片，扩展可能从原 AI 平台或其图片 CDN 拉取图片字节；仅允许可信来源。
- **账号数据最小化**：共享账号时只处理登录 session、用户邮箱、用户 ID、订阅状态和必要的支付状态。
- **不默认建立知识库**：不默认保存完整聊天正文，不默认同步历史聊天列表。
- **权限最小化**：Manifest 只声明导出、登录和下载所需权限。

## 8. 差异化功能设计

本产品的差异化不是“再多支持几个格式”，而是围绕 **可信本地导出、正式交付质量、场景化报告、可验证归档** 建立产品记忆点。

### 8.1 隐私证明面板（Privacy Proof Panel）

目标：让隐私承诺变成用户每次导出前都能看见、能理解、能信任的产品体验。

导出前显示：

- 本次导出将在浏览器本地生成。
- PDF / Word / PNG / Markdown 不会上传到 ChatVault 自有服务器转档。
- 聊天正文不会发送到 Supabase、Analytics 或支付服务。
- 如对话中包含图片，扩展可能从原 AI 平台或其图片 CDN 拉取图片字节。
- 本次导出会消耗几次 Free 额度。
- 本次导出是否会包含原始 URL、平台名、导出时间、角色标签。

产品形式：

- 默认折叠为一行 `Local export verified`。
- 点击后展开完整处理链路。
- 首次使用时默认展开，后续记住用户偏好。

商业化边界：

- Free 和 Pro 都可见基础隐私证明。
- Pro 可附带更完整的 `Export Receipt`。

### 8.2 导出前质量检查（Export Health Check）

目标：在用户点击导出前发现高风险问题，降低“点了没反应”或“导出文件缺图/截断”的失败感。

检查项：

- 消息数量、AI 回复数量、用户 Prompt 数量。
- 是否存在未加载完整的懒加载内容。
- 图片数量、图片来源、图片是否可能跨域失败。
- 是否存在超长代码块、超长表格、超长单词或异常长 URL。
- PNG 是否会超过 canvas 高度，需要自动分段。
- Word / PDF 是否包含表格、代码块、公式、引用等复杂块。
- AI-only 模式下是否会过滤掉所有用户 Prompt。

产品形式：

- 导出按钮上方显示健康状态：`Ready`、`Needs attention`、`High risk`。
- 有风险时给出明确动作，例如 `Scroll to load more messages`、`Split long image export`、`Export as Word instead`。

商业化边界：

- Free 提供基础检查和错误提示。
- Pro 提供更完整检查、修复建议和自动分段策略。

### 8.3 本地敏感信息脱敏（Local Redaction）

目标：让用户放心把 AI 对话变成可分享、可交付文档。

默认识别：

- 邮箱。
- 手机号 / 电话号码。
- 常见 API Key / access token / secret。
- 信用卡格式。
- URL 查询参数中的 `token`、`key`、`secret`、`password`、`session`。
- 常见身份证、税号、护照号格式可作为地区化后续规则。

产品形式：

- 导出设置中提供 `Mask sensitive info` 开关。
- 导出预览中显示“检测到 X 类敏感信息”，默认不展示原文。
- 脱敏样式统一为 `[REDACTED: EMAIL]`、`[REDACTED: API_KEY]`。

商业化边界：

- Free 提供基础脱敏规则。
- Pro 支持自定义规则、保留部分字符、按模板保存脱敏策略。

隐私原则：

- 全部检测在本地完成。
- 不把匹配到的敏感片段写入 Analytics。
- 不默认修改网页内容，只作用于导出副本。

### 8.4 场景化导出模板（Use-case Templates）

目标：用户不只是“导出聊天”，而是直接把聊天变成某类可交付物。

模板优先级：

| 模板 | 适用场景 | MVP |
| :--- | :--- | :---: |
| AI-only Report | 报告、方案、教程、总结 | 是 |
| Q&A Transcript | 复盘、审计、教学 | 是 |
| Research Brief | 研究摘要、竞品分析、资料整理 | P1 / Pro |
| Client Memo | 客户沟通、咨询备忘录 | 后续 |
| Technical Spec | 技术方案、接口设计、架构说明 | 后续 |
| Code Review | 代码审查、修改建议归档 | 后续 |
| Study Notes | 学习笔记、课程整理 | 后续 |
| Legal Memo | 法务/合规留档 | 后续，需谨慎文案 |

模板影响：

- 默认导出标题。
- 是否显示用户 Prompt。
- 是否生成目录。
- 标题层级和章节样式。
- 是否保留 Prompt Appendix。
- 页眉页脚和导出 metadata。

商业化边界：

- Free 提供 Default + AI-only Report。
- Pro 解锁全部专业模板和自定义模板。

### 8.5 AI-only Report 增强版

目标：把 AI 回复整理成更像正式文档的结构，而不是简单删除用户 Prompt。

规则：

- 若 AI 回复中已有 Markdown 标题，保留标题层级并生成目录。
- 若用户 Prompt 很短，可把 Prompt 转为小节标题，正文只保留 AI 回复。
- 多条连续 AI 回复可合并为一个章节。
- 默认不导出原始 Prompt；Pro 可显式开启 `Prompt Appendix`，把原始 Prompt 作为附录放在文末。
- 保留来源 URL、平台、导出时间和可检测到的模型信息。

MVP 不调用外部 AI 重新总结，只做本地结构化和格式整理，避免破坏“本地生成”承诺。

### 8.6 可验证导出凭证（Export Receipt）

目标：服务研究、工作归档、法务、审计和需要可追溯记录的用户。

Receipt 内容：

- 导出时间。
- 平台。
- 原始 URL。
- 导出格式。
- 导出模式。
- 消息数量。
- 是否启用脱敏。
- 是否本地生成。
- 文件 SHA-256 hash。
- 扩展版本。

产品形式：

- 可嵌入 PDF / Word 文末。
- 可作为 `.json` sidecar 文件另存。
- 可在导出完成 Toast 中展示 `Copy receipt`。

商业化边界：

- Free 显示基础 metadata。
- Pro 支持完整 Receipt、sidecar JSON、文件 hash。

### 8.7 开发者专用导出模式（Developer Export）

目标：对开发者形成明确吸引力，避免只做泛办公导出。

能力：

- 按语言汇总代码块。
- 生成代码块索引。
- Word / PDF 中保留代码预览和上下文说明。
- Pro 支持把代码块另存为 `.js`、`.py`、`.ts`、`.md` 等文件并打包 zip。
- 可选导出 `README.md`，列出每段代码来自哪条 AI 回复。

MVP：

- 保留代码高亮和语言标识。
- 生成代码块索引。

后续 Pro：

- 代码 zip 打包。
- 按语言过滤导出。

### 8.8 社交分享卡片与分镜长图（Share Cards）

目标：让导出结果不仅能归档，也能传播。

能力：

- 单条回答 Quote Card。
- X / Twitter thread cards。
- LinkedIn carousel。
- 9:16 story cards。
- 小红书长图比例。
- 超长内容自动切片，并保持页码和连续视觉风格。

商业化边界：

- Free 提供基础单条回答卡片。
- Pro 提供高分辨率、无水印、平台比例模板和批量切片。

## 9. MVP 验收标准

- ChatGPT / Claude / Gemini 当前会话可成功导出 Word、PDF、PNG、Markdown。
- Free 用户免登录可完成 3 次基础导出。
- 已登录 Pro 用户可使用高级主题、隐藏水印和高级设置。
- 导出前显示隐私证明面板，明确说明全格式本地生成。
- 导出前质量检查可识别至少消息数量、图片数量、PNG 分段风险和未加载内容风险。
- AI-only Report 可在不调用外部 AI 的情况下生成可读报告结构。
- 基础敏感信息脱敏可识别邮箱、电话、API key 和敏感 URL 参数。
- 导出失败、取消、超长、图片失败都有明确反馈。
- 不读写 `chat_indexes`、`folders`、`folder_chats`、`bookmarks` 等主产品管理表。
- 导出模块复用主产品已有 fixture 测试，避免图片丢失、顺序错乱、内部 prompt 泄露等回归。
